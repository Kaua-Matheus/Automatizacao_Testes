# Testes

## Testes E2E
- Criar pelo menos 2 testes E2E que cubram fluxos de usuário completos (ex: fluxo de cadastro, fluxo de login e curtida com e sem autenticação etc.).

## Testes API
- Criar pelo menos 4 testes de API (caixa preta), validando o comportamento dos endpoints com entradas conhecidas (ex: POST /auth/signin com sucesso, POST /auth/signup com e-mail duplicado etc.).
